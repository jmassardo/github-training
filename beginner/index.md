## Phase Overview

Welcome to the beginning of your GitHub journey! This phase establishes the foundation you need to understand how software development works and how GitHub fits into that process.

### What You'll Learn

By the end of this phase, you'll understand:

- How software teams plan, develop, and collaborate
- The Software Development Lifecycle (SDLC) and DevOps principles
- Git version control fundamentals and workflows
- GitHub.com platform features and navigation
- Source Code Management (SCM) as a team process
- Planning and collaboration tools (Issues, Projects, PRs)

### Phase Context

**Before:** You're an experienced CSM with strong customer skills but limited software development exposure.

**After:** You'll speak confidently about software development processes, navigate GitHub with ease, and guide customers through basic workflows.

### üìò Module 1: Understanding Software Development & SDLC
**Duration:** 8-10 hours | **Weeks:** 1

Before diving into tools, understand the problem space: How do software teams work? What problems does GitHub solve?

**Topics:**

## Beginner Phase Milestone Project

### üéØ Project: Collaborative Documentation Repository

**Objective:** Demonstrate mastery of beginner concepts by building a collaborative documentation repository for a fictional product team.

**Requirements:**

1. Create a repository with proper README, CONTRIBUTING, and CODE_OF_CONDUCT
2. Set up branch protection rules requiring PR reviews
3. Create a GitHub Project board with columns for documentation tasks
4. Open 5+ issues with proper labels, assignments, and descriptions
5. Submit 3+ pull requests with detailed descriptions and resolve review comments
6. Implement CODEOWNERS for different documentation sections
7. Create a release with changelog and semantic versioning
8. Document the workflow in a wiki or GitHub Pages site

**Deliverables:**

- Live repository demonstrating all requirements
- 10-minute presentation walking through your workflow
- Written reflection on lessons learned

**Assessment Criteria:**

- Repository structure and documentation quality (25%)
- Proper use of Issues and Projects (25%)
- PR process and code review interactions (25%)
- Branch protection and release management (15%)
- Presentation clarity and insights (10%)
