## How Teams Use GitHub

### Small Team (5-10 developers)
- Single repository or few repos
- GitHub Flow branching
- Basic code review
- Free or Team tier

### Mid-Size Company (50-200 developers)
- Multiple repositories
- Team permissions and CODEOWNERS
- GitHub Projects for planning
- Actions for CI/CD
- Enterprise Cloud

### Enterprise (1000+ developers)
- Organization hierarchy
- SSO/SAML integration
- GitHub Advanced Security
- Compliance requirements
- Enterprise Cloud or Server
