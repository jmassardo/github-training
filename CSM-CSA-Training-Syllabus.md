## Program Overview

This 12-week progressive training program is designed specifically for **experienced Customer Success Managers** transitioning into GitHub CSM/CSA roles. The curriculum assumes strong customer-facing skills but no prior software development or GitHub experience.

**Target Audience:**
- Experienced CSMs new to GitHub
- CSAs transitioning from other platforms
- Technical account managers entering DevOps space
- Customer success professionals upskilling for developer tools

**Learning Philosophy:**
The syllabus follows the **natural software development workflow**, introducing concepts as developers encounter them:
1. Understanding the problem space (planning & requirements)
2. Writing and organizing code (development & collaboration)
3. Quality assurance (testing & security)
4. Delivery (CI/CD & deployment)
5. Operations (monitoring & maintenance)
6. Enterprise governance (administration & compliance)

**Learning Objectives:**
- Understand software development workflows and SDLC phases
- Master GitHub's role in each development stage
- Navigate GitHub platform with confidence
- Guide customers through GitHub adoption journeys
- Implement security, compliance, and governance best practices
- Measure and improve DevOps performance with DORA metrics
- Communicate technical concepts to diverse audiences

**Time Commitment:** 15-20 hours per week (includes hands-on labs, reading, and practice)
